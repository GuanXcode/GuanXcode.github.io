# Agent任务编排的最小闭环

很多团队在做 Agent 时会先追求“能力全”，结果是难以验证与交付。我更推荐先搭建最小闭环：

## 闭环结构

1. 意图识别：明确任务目标与约束
2. 工具选择：只保留必要工具，降低执行分支
3. 执行反馈：每步输出可观测日志
4. 结果评估：基于规则与样例双校验

## 关键原则

- 先可解释，再自动化
- 先稳定，再扩展
- 先闭环，再并行

这套方法的价值在于：即使模型能力波动，系统依然能通过流程约束保持可控。
